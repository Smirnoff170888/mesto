По организации класса Api и использованию его методов все очень круто:
- реализован весь функционал из задания
- используется async/await
- страница изменяется только после ответа сервера

Но есть проблема - иногда при загрузке страницы скрипт падает с ошибкой:
script.js:104 Uncaught (in promise) ReferenceError: Api is not defined
    at HTMLDocument.<anonymous> (script.js:104)

Это происходит из за, того, что скрипты подключены c атрибутом async.
Загружая скрипт асинхронно, вы сообщаете браузеру, что он может загрузить 
этот скрипт независимо от других частей страницы. Это означает, что страница
может закончить загрузку и DOMContentLoaded может сработать до того, как загрузится ваш скрипт.

async следует убрать. В следующих проектных работах Вы научитесь собирать скрипты в один бандл
и подключать только его.
